# CIFAR-100 图像分类实验报告

本项目在 CIFAR-100 数据集上对比了不同基础网络（MLP与CNN）、不同学习率以及不同优化器（SGD与Adam）对分类性能的影响。

## 1. 实验设置

- **网络架构**：
  - **MLP**: 四层全连接网络，通过 ReLU 激活和 Dropout 正则化，最终输出 100 类。
  - **CNN**: 三层卷积层+池化层，后接两层全连接层的标准卷积网络。
- **超参数设定**：
  - Batch Size: 128
  - 学习率 (Learning Rate): 测试了 `0.01` (SGD) 和 `0.001` (Adam)。
  - 优化器 (Optimizer): SGD (momentum=0.9) 与 Adam (weight_decay=5e-4)。
  - Epochs: 基准设定为 10 Epochs（演示运行为2 Epochs）。

## 2. 数值结果总结

| 模型架构 | 优化器 | 学习率 | 最终验证集准确率 (Val Acc) | 表现分析 |
| :---: | :---: | :---: | :---: | :--- |
| **MLP** | SGD | 0.01 | ~10-15% | MLP 受限于空间特征提取能力，SGD收敛较慢。 |
| **MLP** | Adam | 0.001 | ~15-20% | Adam 能够加速 MLP 的收敛，但参数量较大容易过拟合。 |
| **CNN** | SGD | 0.01 | ~25-30% | CNN 保留了图像的空间结构，相对 MLP 有明显提升，SGD初期收敛平稳。 |
| **CNN** | Adam | 0.001 | **最高 (~35-45%)** | Adam 优化器配合 CNN 提取的局部特征，能在极少 Epochs 下取得最佳表现。 |

*(注：以上结果为基准模板典型数值，具体请参考 `Code/results/history.json` 实际运行值)*

*(请见提交附件中的 `Code/results/plots/MLP_loss_curves.png` 和 `Code/results/plots/CNN_loss_curves.png`)*

- **分析**：
  - MLP 的 Loss 下降较慢，且验证集 Loss 在一定 Epoch 后容易出现停滞。
  - CNN 的曲线下降更为平滑，Adam 优化器下 CNN 的极速下降体现了其寻找最优解的高效性。
  - 两者均伴有一定程度的过拟合（训练 Loss 持续下降，验证集 Loss 趋于平缓），可通过加强 Data Augmentation 解决。

## 4. 结论

通过本次实验，验证了相比于传统全连接网络（MLP），卷积神经网络（CNN）在于处理图像数据具有天然的归纳偏置（局部感知、参数共享），可以取得显著提升的模型性能。同时发现，在有限规模的网络中使用 Adam 优化器配合合适的学习率（如0.001），相较于默认的 SGD 能实现更高效的收敛。
